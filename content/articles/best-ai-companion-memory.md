### Understanding Different Types of AI Memory for Agents

Effective AI companion memory isn't a monolithic concept. It's a combination of different memory types, each serving a distinct purpose. #### Episodic Memory in AI Agents

**Episodic memory** allows an AI agent to store and recall specific events or experiences, much like human autobiographical memory. For an AI companion, this means remembering distinct conversations, user actions, or specific moments in time. This is vital for recalling past interactions with detail, a key component of the **best AI companion memory**.

This type of memory enables an AI to say, "Last Tuesday, you mentioned you were planning a trip to the mountains. Did you decide on a destination?" instead of a generic "How can I help you today?" It provides a sense of continuity and personal connection. The ability to retrieve these specific "episodes" is a hallmark of advanced AI companions and central to the **best AI companion memory**. #### Semantic Memory in AI Agents

**Semantic memory** stores general knowledge, facts, concepts, and language meaning. For an AI companion, this includes understanding the world, common sense, and the definitions of words. It's the AI's foundational knowledge base, supporting the overall functionality of its memory and contributing to a **robust AI companion memory**.

This memory type ensures the AI can answer factual questions, explain concepts, and use language correctly. It's the bedrock upon which more dynamic memories are built. Without strong semantic memory, an AI companion would struggle to understand basic queries or provide accurate information. Explore the role of [semantic memory in AI agents](/articles/semantic-memory-ai-agents/).

#### Short-Term vs. Long-Term Memory for AI

AI companions use both **short-term memory** (STM) and **long-term memory** (LTM). STM, often referred to as the **context window** in Large Language Models (LLMs), holds information currently being processed, typically for a limited duration. LTM stores information persistently over extended periods, forming the basis for recall in the **best AI companion memory** systems.

LLMs often face **context window limitations** that restrict how much information they can actively process at once. Advanced **AI companion memory systems** must effectively bridge this gap, transferring relevant information from STM to LTM and efficiently retrieving it when needed. This is a significant challenge addressed by many modern [LLM memory systems](/articles/llm-memory-system/), crucial for effective **AI recall**.

import datetime

class AICompanionMemory:
 def __init__(self, max_log_size=20, similarity_threshold=0.7):
 # Using a list to store conversation turns chronologically
 # Each turn is a dictionary with 'timestamp', 'speaker', and 'utterance'
 self.memory_log = []
 self.max_log_size = max_log_size # Limit memory to last N turns
 self.similarity_threshold = similarity_threshold # For simulated semantic search

 def add_memory(self, speaker, utterance):
 """Adds a new memory entry with a timestamp."""
 self.memory_log.append({
 "timestamp": datetime.datetime.now(),
 "speaker": speaker,
 "utterance": utterance
 })
 # Trim memory if it exceeds the maximum size
 if len(self.memory_log) > self.max_log_size:
 self.memory_log.pop(0) # Remove the oldest entry

 def retrieve_last_utterance(self):
 """Retrieves the last utterance from the memory log."""
 if not self.memory_log:
 return "I don't have any prior conversation to recall."
 last_entry = self.memory_log[-1]
 return f"{last_entry['speaker']} ({last_entry['timestamp'].strftime('%H:%M')}): {last_entry['utterance']}"

 def retrieve_context(self, num_turns=3):
 """Retrieves the last 'num_turns' of conversation."""
 if not self.memory_log:
 return "No conversation history available."
 # Return the most recent turns, ensuring not to exceed available logs
 start_index = max(0, len(self.memory_log) - num_turns)
 context = self.memory_log[start_index:]
 formatted_context = "\n".join([f"{entry['speaker']} ({entry['timestamp'].strftime('%H:%M')}): {entry['utterance']}" for entry in context])
 return formatted_context

 def find_similar_memories(self, query_utterance, num_results=2):
 """
 Simulates finding semantically similar memories.
 In a real system, this would use embeddings and a vector database.
 Here, we'll use a simple keyword overlap as a proxy.
 """
 if not self.memory_log:
 return "No memories to search."

 query_words = set(query_utterance.lower().split())
 scored_memories = []

 for entry in reversed(self.memory_log): # Search recent first
 entry_words = set(entry['utterance'].lower().split())
 common_words = query_words.intersection(entry_words)
 score = len(common_words) / len(query_words) if query_words else 0

 if score >= self.similarity_threshold:
 scored_memories.append((score, entry))

 # Sort by score (descending) and take top N results
 scored_memories.sort(key=lambda x: x[0], reverse=True)
 top_results = scored_memories[:num_results]

 if not top_results:
 return "No highly relevant past memories found."

 formatted_results = "\n".join([
 f" - Score: {score:.2f}, {entry['speaker']} ({entry['timestamp'].strftime('%Y-%m-%d %H:%M')}): {entry['utterance']}"
 for score, entry in top_results
 ])
 return f"Found {len(top_results)} relevant memories:\n{formatted_results}"
